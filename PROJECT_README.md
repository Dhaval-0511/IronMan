# MARK — HUD Systems

A scroll-driven 3D-feeling website: sticky-canvas frame-sequence hero, a tunnel-style
"Arsenal" showcase, a bento systems grid, a flight log, live diagnostic gauges, and a CTA —
built with Next.js 16, React 19, Tailwind v4, Framer Motion, and Lenis smooth scroll.

## Running it

```bash
npm install
npm run dev
```

Then open http://localhost:3000

To build for production:

```bash
npm run build
npm run start
```

## What's inside

- `src/app/layout.tsx` — root layout, fonts, smooth-scroll provider
- `src/app/page.tsx` — composes every section
- `src/components/sections/Hero.tsx` — sticky canvas that scrubs through
  `public/frames/frame_0001.jpg … frame_0106.jpg` as you scroll (400vh scroll track)
- `src/components/sections/ArsenalShowcase.tsx` — same technique with
  `public/tunnel-frames/frame_0001.jpg … frame_0097.jpg` (500vh scroll track), plus
  5 annotation cards and an end-of-scroll CTA overlay
- `src/components/sections/Systems.tsx` — bento grid of suit subsystems
- `src/components/sections/FlightLog.tsx` — timestamped mission log
- `src/components/sections/Diagnostics.tsx` — animated SVG gauge cluster
- `src/components/sections/FinalCta.tsx`, `Footer.tsx`
- `src/components/providers/SmoothScrollProvider.tsx` — Lenis wrapper
- `src/components/ui/` — shared Button, EyebrowBadge, AnimatedSection primitives

## Frame sequences

The two frame folders (`public/frames` and `public/tunnel-frames`) were generated by
extracting frames from the two Kling-generated videos you supplied
(`kling_..._Cinematic_...mp4` → hero, `kling_..._Futuristic_...mp4` → showcase) with:

```bash
ffmpeg -i input.mp4 -vf "scale=1600:-2,fps=<count>/<duration>" -q:v 3 public/frames/frame_%04d.jpg
```

If you want to swap in a different clip, re-run that command with your own video and the
same `frame_XXXX.jpg` naming — the components read `FRAME_COUNT` from the top of each file,
so update that constant to match how many frames you export.
